Commands

Slash commands available in superbot sessions. All commands are prefixed with superbot:.

Reference

Command Description
/superbot:slack-setupSet up the Slack bot integration
/superbot:onboardRun interactive onboarding
/superbot:statusShow overview of superbot state
/superbot:skillsBrowse, install, and manage skills
/superbot:restartResume session with fresh context
/superbot:doctorDiagnose and fix common issues

/superbot:slack-setup

Set up the Slack bot integration. This is the recommended way to communicate with superbot from your phone or desktop.

Walks through creating a Slack app from a manifest, collecting tokens, saving config, testing the connection, and installing the daemon. See Slack Integration for the full guide.

Allowed tools Bash, Read, Write, Edit

/superbot:onboard

Interactive onboarding to configure your identity and preferences. Automatically launched by the setup script on first run.

Walks through naming your bot, defining its personality, learning about you, and installing optional skills.

Allowed tools Bash, Read, Write, Edit

/superbot:status

Show an overview of all superbot state — identity, memory, pending tasks, daily notes, and heartbeat activity.

Allowed tools Read, Bash, Glob

/superbot:skills

Browse, install, and manage skills and integrations.

Examples

/superbot:skills list
/superbot:skills install gog
/superbot:skills search slack
/superbot:skills remove frontend-design

Allowed tools Bash, Read, Write, Edit, Glob, Grep, WebFetch

/superbot:restart

Resume the current session with freshly assembled context. Re-reads all files, reloads skills, and restarts background daemons. The conversation history is preserved via --resume.

Allowed tools Bash

/superbot:doctor

Diagnose and fix common issues. Runs the infrastructure doctor script first (CLI, auth, config, launchd), then checks file contents for health problems.

What it checks

  • MEMORY.md size — warns if growing too large for the context window
  • IDENTITY.md and USER.md size — these should stay concise
  • Stale completed tasks in HEARTBEAT.md
  • Daily notes accumulation — old notes piling up
  • Observer health — stale offset or missing offset file

After reporting, offers to auto-fix what it can (clear completed tasks, delete stale files) and advises on things that need human judgment (pruning memory, archiving old notes).

Allowed tools Read, Write, Edit, Bash, Glob
