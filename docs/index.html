Persistent identity and background automation for Claude Code. Memory that survives sessions, a heartbeat that works while you sleep, and a personality that's actually yours.

Why Superbot

Claude Code is powerful, but every session starts from zero. It doesn't know your name. It doesn't remember what you worked on yesterday. It can't do anything while you're away. And it has no personality beyond whatever the base model gives you.

Superbot fixes that. It's a Claude Code plugin that gives your assistant continuity — a persistent identity, long-term memory, daily notes that write themselves, and background workers that process tasks while you sleep. You configure it once, and it remembers everything from there.

The result is an assistant that knows who you are, what you're working on, and what you care about. Not because you told it five minutes ago, but because it's been paying attention across every session.

Spin it up on a spare machine, run caffeinate, and you have an assistant that's always on. Message it over Slack from your phone at 2am. Ask it to check your email in the morning. Queue up tasks before bed and wake up to results. It's not a chatbot you visit — it's an assistant that's waiting for you.

Architecture

How system prompt assembly, the launcher, team messaging, and file layout all fit together.

How It Works

Superbot wraps Claude Code with a launcher that assembles a system prompt from live context files every session. Your identity, memory, pending tasks, and daily notes are all injected at startup. When you restart, the launcher re-reads everything fresh — that's the continuity mechanism.

In the background, a heartbeat worker checks for pending work orders every 30 minutes. A triage model decides if real work is needed. If so, workers are spawned as persistent Claude sessions — each with its own session ID that can be resumed across messages. For Slack, superbot acts as a pure router: simple messages get direct replies, everything else gets forwarded to a worker via spawn-worker.sh. Workers do the actual work and report results back to superbot's inbox. A session observer parses your conversations and writes key events to daily notes automatically.

Everything is plain files. Markdown for memory, markdown checkboxes for tasks, JSON for configuration. No database, no server, no build step. You can read and edit every piece of state with a text editor.
