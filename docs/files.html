File Reference

Every config file, what it does, and where it lives.

User Config Files

Location: ~/.superbot/

File Purpose Updated By
config.json Settings — models, Slack tokens, schedule, heartbeat config. Created from config.template.json. Setup, user
IDENTITY.md Bot personality — name, communication style, proactivity level, guiding principles User, onboarding
USER.md User profile — name, about, common tasks, interests User, onboarding
MEMORY.md Persistent memories and learnings. Curated long-term knowledge base. User, Claude, heartbeat worker
HEARTBEAT.md Task queue. Pending (- [ ]) and completed (- [x]) tasks with notes. User, heartbeat worker
daily/YYYY-MM-DD.md Daily notes — auto-populated history of each day's work. Session observer, heartbeat worker, manual
sessions.json Worker session registry — tracks all spawned workers, their Slack thread mappings, space associations, and status. spawn-worker.sh
prompts/heartbeat.md Heartbeat worker instructions. Customizable copy of scripts/worker-prompt.md. Setup, user
.observer-offset Tracks observer progress — session ID and line offset for incremental parsing. Session observer (daily-observer.sh)

Slack Files

Location: ~/.superbot/

File Purpose
slack-config.json Legacy Slack bot tokens. Migrated to config.json on setup.
logs/slack-bot.log Application-level Slack bot logs (messages, sessions, errors).
logs/slack-stdout.log Launchd stdout for the Slack daemon.
logs/slack-stderr.log Launchd stderr for the Slack daemon.

Scheduler Files

Location: ~/.superbot/

File Purpose
schedule-last-run.json Dedup tracking. Maps job names to their last execution key.
logs/scheduler.log Scheduler activity: running, completed, failed jobs.
logs/scheduler-stdout.log Launchd stdout for the scheduler daemon.
logs/scheduler-stderr.log Launchd stderr for the scheduler daemon.

Heartbeat Logs

Location: ~/.superbot/

File Purpose
logs/heartbeat.log Triage decisions, worker spawning, completion events.
logs/heartbeat-stdout.log Launchd stdout for the heartbeat cron.
logs/heartbeat-stderr.log Launchd stderr for the heartbeat cron.

Team Files

Location: ~/.claude/teams/superbot/

File Purpose
config.json Team configuration — name, description, lead agent/session IDs, member list.
inboxes/superbot.json Team lead inbox. Receives messages from heartbeat worker and Slack bot.
inboxes/heartbeat.json Heartbeat worker inbox. Receives kickoff messages from cron script.

Repository Template Files

Location: ~/superbot/templates/

File Purpose
SYSTEM.md Core system instructions — behaviors, file purposes, commands, guidelines.
IDENTITY.md Template for bot persona. Copied to user config on setup.
USER.md Template for user profile. Copied to user config on setup.
MEMORY.md Template for persistent memory. Copied to user config on setup.
HEARTBEAT.md Template for task queue. Copied to user config on setup.
ONBOARD.md Onboarding conversation script. Deleted after onboarding completes.
CLAUDE_SYSTEM_PROMPT.md Base Claude behavior instructions included in system prompt.

Plugin & Config Files

Location: ~/superbot/

File Purpose
.claude-plugin/plugin.json Plugin metadata — name, version, description, author, keywords.
hooks/hooks.json Session event hooks. Currently empty, reserved for future use.
package.json Node.js dependencies. Currently just @slack/bolt.
scripts/slack-manifest.json Slack app manifest for creating the bot via the Slack API.

Launchd Plists

Location: ~/Library/LaunchAgents/

Plist Service Interval
com.claude.superbot-heartbeat.plist Heartbeat cron — runs heartbeat-cron.sh Every 1800s (30 min)
com.claude.superbot-scheduler.plist Scheduler — runs scheduler.sh Every 60s
com.claude.superbot-slack.plist Slack bot — runs slack-bot.js KeepAlive (continuous)

Scripts Reference

Location: ~/superbot/scripts/

Script Purpose
setup.sh Full initialization — create directories, copy templates, set up team, shell profile
heartbeat-cron.sh Background worker spawner — triage, context assembly, spawn teammate
triage.sh Lightweight work detector — Haiku model answers YES/NO
triage-prompt.md Instructions for the triage agent
spawn-worker.sh Unified worker spawner — creates/resumes persistent sessions for Slack and space workers
slack-worker-prompt.md System prompt for Slack workers spawned by spawn-worker.sh
space-worker-prompt.md System prompt for space workers (used with --space flag)
worker-prompt.md Instructions for the heartbeat worker teammate
daily-observer.sh Session observer — parses active session, appends key events to daily notes
parse-session.sh JSONL parser — extracts conversational content from session files
observer-prompt.md Instructions for the Haiku observer model
scheduler.sh Cron-style job executor — reads schedule array from config.json, spawns Claude for due jobs
slack-bot.js Full Slack bot — Socket Mode, DMs, @mentions, thread continuity
slack-manifest.json Slack app configuration manifest
inject-context.sh Context injection helper for other scripts
install-heartbeat.sh Install heartbeat background service
install-scheduler.sh Install scheduler launchd job
install-slack.sh Install Slack daemon launchd job
uninstall-heartbeat.sh Remove heartbeat background service
uninstall-slack.sh Remove Slack daemon
status.sh Quick status check from the command line
doctor.sh Comprehensive diagnostics and health check
test-triage.sh Test triage decision without spawning worker
test-worker.sh Dry-run worker with read-only tools
