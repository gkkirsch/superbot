Scheduler

Time-based cron jobs for recurring tasks.

Heartbeat vs Scheduler

Superbot has two background systems. Use the right one:

Need Use
"Do this whenever you can"Heartbeat
"Do this at 7am every weekday"Scheduler
"Remind me about X tomorrow morning"Scheduler
"Look into this and get back to me"Heartbeat
"Check my email every morning"Scheduler
"Run tests on the main branch"Heartbeat

The heartbeat is a task queue — "do this when you get a chance." The scheduler is a cron system — "do this at a specific time."

How It Works

The scheduler daemon checks every 60 seconds if any jobs are due. When a job's time matches, it spawns a Claude instance with the task and full context (IDENTITY, USER, MEMORY).

  1. Reads the schedule array from config.json for job definitions
  2. Checks current time and day of week against each job
  3. Checks schedule-last-run.json to avoid running the same job twice
  4. For each due job, spawns Claude (Opus by default, configurable via scheduler.model) with the task
  5. Logs results to ~/.superbot/logs/scheduler.log

Schedule Format

Jobs are defined in the schedule array in config.json:

{
  "schedule": [
    {
      "name": "morning-briefing",
      "time": "07:00",
      "days": ["mon", "tue", "wed", "thu", "fri"],
      "task": "Summarize my email inbox and calendar for today"
    },
    {
      "name": "weekly-status",
      "time": "17:00",
      "days": ["fri"],
      "task": "Write a weekly status report based on this week's daily notes"
    },
    {
      "name": "daily-cleanup",
      "time": "23:00",
      "task": "Review today's daily notes and promote anything important to MEMORY.md"
    }
  ]
}

Fields

Field Required Description
name Yes Unique identifier for the job (used for dedup)
time Yes 24-hour format HH:MM
days No Array of day abbreviations. Omit for every day.
task Yes What Claude should do when the job fires

Day abbreviations

mon, tue, wed, thu, fri, sat, sun

Examples

Morning briefing (weekdays at 7am)

{
  "name": "morning-briefing",
  "time": "07:00",
  "days": ["mon", "tue", "wed", "thu", "fri"],
  "task": "Summarize my email inbox and calendar for today"
}

Weekly report (Fridays at 5pm)

{
  "name": "weekly-report",
  "time": "17:00",
  "days": ["fri"],
  "task": "Write a weekly status report based on this week's sessions and daily notes"
}

Daily cleanup (every night at 11pm)

{
  "name": "nightly-cleanup",
  "time": "23:00",
  "task": "Review today's daily notes and promote important items to MEMORY.md"
}

Installing the Scheduler

~/superbot/scripts/install-scheduler.sh

This creates a launchd plist at ~/Library/LaunchAgents/com.claude.superbot-scheduler.plist that runs every 60 seconds.

Deduplication

The scheduler tracks what it ran in ~/.superbot/schedule-last-run.json. Each job is keyed by name:date:time, so it won't fire the same job twice in the same minute. The file looks like:

{
  "morning-briefing": "morning-briefing:2026-02-06T07:00",
  "weekly-report": "weekly-report:2026-02-06T17:00"
}

Log Contents
~/.superbot/logs/scheduler.log Job execution: running, completed, failed
